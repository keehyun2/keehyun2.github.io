DDD START(ch.2)

아키텍처 소프트웨어 아키텍처 에는 4가지 영역(계층)이 있습니다. (표현, 응용, 도메인, 인프라) 표현 영역(Controler)는 http request 와 response를 처리하는 계층입니다. 응용 영역(Service)는 내부 로직을 구현합니다. 로직을 직접 구현하기보단 도메인에 로직 구현을 위임합니다. 도메인 영역(Order, DTO, VO)은 도메인 모델을 구현하고, 도메인과 관련된 핵심 로직을 구현합니다. 인프라스트럭처 영역 (jpa 를 사용한 경우 Repository) 은 구현 기술을 다룹니다. 다른 ORM 프레임워크나 외부 시스템과 연동을 지원합니다. (계층구조) 아키텍처 4가지 영역을 사용할때 보편적으로 계층구조 아키텍처를 사용합니다....
